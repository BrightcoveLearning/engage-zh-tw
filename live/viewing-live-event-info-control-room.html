使用控制室查看實時事件信息

在本主題中，您將學習如何使用控制室查看實時事件信息。

“控制室"頁面用於查看有關現場活動的信息。當您創建新的直播事件並開始流式傳輸時，“控制室"頁面將打開，顯示事件信息。事件的預覽將出現在預覽播放器中。當您單擊“管理直播事件"頁面上的事件時，也會顯示“控制室"頁面。

控制室頁面活動信息標籤分為以下幾部分：

  • 活動信息
  • 流信息
  • 短片
  • 監視器

查看活動信息

這活動信息部分顯示以下信息：

  • 活動名稱 -活動名稱
  • 簡短的介紹 -事件的簡短說明
  • 標籤 -與事件關聯的標籤。標籤有助於搜索，可以在創建智能播放列表時使用。
  • 參考編號 -事件的內部參考ID
  • 流媒體到社交媒體 - 如果允許的，直播活動可以分配給預定的社交活動
  • 數字版權管理（DRM） --
  • 播放網址 -可用於在非Brightcove播放器中觀看直播的URL

查看流信息

這流信息部分顯示以下信息：

  • 工作編號 -內部工作編號
  • 影片編號 -直播事件流的視頻ID（這也將是“視頻"頁面上VOD資產的視頻ID）
  • Endpoint (RTMP URL) - Streaming endpoint used by the encoder
  • Name - Stream name used by the encoder (always alive)
  • 地區 -流域
  • 地位 -當前工作狀態
    • 流媒體 - 直播
    • 處理錯誤 - 直播遇到錯誤
    • 支持 - 流準備好被激活（僅限靜態入口點）
    • 停止 - 直播正在停止並創建任何相關的 VOD 輸出
    • 完全的 - 直播已完成
    • 取消 - 直播正在停止，不會處理任何掛起的 VOD 輸出
    • 取消 - 直播已取消
    • 斷開連接 - 編碼器已斷開
    • 等待 - 等待 RTMP 流（編碼器啟動）
    • 失敗的 - 由於系統錯誤，作業已停止

觀看短片

這短片部分顯示從實時流創建的所有剪輯（如果有）。單擊片段名稱將在屏幕上打開片段。影片頁。有關創建剪輯的信息，請參見創建視頻剪輯。

監控流

這監視器部分顯示輸出和輸入漂移圖。點擊檢索更新的數據和圖形鏈接以查看信息。輸出漂移可衡量河流的健康狀況。輸出漂移定義為牆上時鐘與流時間戳（TS）之間的差。理想情況下，在實時流傳輸中，掛鐘應以與流TS相同的速度運行，如果流TS的到達速度比實時速度慢，則意味著系統存在問題（瓶頸）。

輸入漂移測量輸入流的運行狀況，並定義為與輸入時間戳和實時時鐘之差。
