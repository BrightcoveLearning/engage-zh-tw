創建和管理現場活動

在本主題中，您將學習如何創建和管理實時事件。

介紹

Brightcove Engage 中的實時視頻功能™可用於向移動應用程序和 Web 體驗廣播實時事件。創建現場活動後，將為您提供一組編碼器設置，您可以將其插入到現場編碼器中。所有轉碼都在雲中完成，因此您不需要基於硬件的轉碼器。然後，在創建“移動應用程序"和“Web體驗"時，可以包含實況視頻。

先決條件

您必須播出以下項目，然後才能廣播直播事件：

  • 攝像機播放事件
  • 您必須擁有自己的編碼軟件，例如Telestream的有線廣播或者開放廣播軟件

限制

  • 直播將延遲20-30秒
  • 直播必須在活動開始後30分鐘內開始
  • 對於使用Internet Explorer的用戶，需要版本10或更高版本

管理現場活動

要開始直播活動，請登錄Brightcove 參與™然後點擊居住在導航標題中。“管理事件"頁面顯示兩個主要選項卡：

  • 居住 -列出正在進行的現場活動
  • 完全的 -列出已完成的事件

這表演下拉菜單可用於過濾出現在完全的標籤。

該頁面的正文將顯示實時事件的列表以及相關信息：

  • 作業狀態-實時流的狀態
    • 加工 -直播
    • 處理錯誤 -直播流遇到錯誤
    • 支持 -準備激活的流（僅靜態入口點）
    • 停止中 -實時流正在停止並創建任何關聯的VOD輸出
    • 完全的 -直播已完成
    • 取消 -實時流正在停止，不會處理任何待處理的VOD輸出
    • 取消 -直播已取消
    • 斷線 -編碼器已斷開連接
    • 等待 -等待RTMP流（編碼器啟動）
    • 失敗的 -由於系統錯誤，作業已停止
    • 棄 -未連接編碼器的事件結束
  • 完成日期和時間
  • 活動名稱和ID
  • 總瀏覽
  • 持續時間
  • 創建的日期和時間（僅適用於重複事件）

要查看現場直播視頻屬性，請點擊事件名稱以打開控制室。

創建一個新事件

要創建一個新的現場活動，請按照下列步驟操作。

  1. 在“管理事件"頁面中，單擊建立活動。這創建實時事件頁面將打開。
  2. 輸入事件詳細信息：
    • 活動名稱 -活動名稱；如果將視頻另存為視頻資產，此名稱也將出現在“視頻"頁面中（最多255個字符）
    • 簡短的介紹 - 事件描述（最多 250 個字符）
    • 標籤 -與事件關聯的標籤；如果將視頻另存為視頻資產，則會將標籤分配給視頻（最多128個字符和1200個標籤）
    • 參考編號 -事件的參考ID；必須是唯一的（最多150個字符）
  3. 選擇完成後將事件轉換為視頻資產將實時視頻的視頻點播版本保存到您的帳戶。請注意，直播活動結束後將無法使用VOD版本，上傳過程完成後，視頻將可用。
  4. 在裡面流選項部分，選擇一個地區。這地區是要使用的Amazon Web Services（AWS）區域。為了減少延遲，您應該選擇最接近編碼器所在的區域。默認情況下，該地區設置為俄勒岡州，如果您是從其他大洲流媒體，則會產生不好的結果。支持以下區域：
    • 俄勒岡州（默認）
    • 弗吉尼亞州
    • 東京
    • 新加坡
    • 悉尼
    • 孟買
    • 法蘭克福
    • 愛爾蘭
  5. 點擊高級選項和配置高級選項。
  6. 點擊建立活動開始直播活動。這控制室頁面將打開有關直播活動的詳細信息，包括流式傳輸端點（RTMP URL）和流名稱將由您的編碼器使用。
    注意：處理您的流大約需要一分鐘。

    默認情況下，編碼器必須在創建新事件後 30 分鐘內啟動。

點擊停止串流按鈕結束活動。

注意：直播結束後，請確保關閉/停止編碼器。使任何編碼器和 Brightcove 之間的連接保持打開狀態，即使沒有發送數據，也可能導致意外的連接費用。

配置高級選項

創建新事件時，可以配置高級選項。要配置高級選項，請單擊高級選項創建事件時鏈接。

可以配置以下選項：

  • 內容加密 -選擇要應用於實時流的加密類型：
    • 沒有任何 -未應用加密
    • AES-128 -使用AES-128加密保護流。如果選擇此選項，則必須輸入您的加密金鑰。密鑰必須是32個十六進製字符。為了獲得最佳效果，請使用密鑰生成器，選擇128-bit和Hex選項。
      無法從加密的實時流中創建剪輯。
  • 輸入格式 -來自編碼器的輸入格式，將被發送到Brightcove Live。此功能可在實時流（如現在，現在或下一個）中啟用其他元數據，從而在字幕和字幕，音軌方面具有更大的靈活性，並可以使用標準廣播級數據簡化數字體驗。支持以下輸入格式：
    • RTMP（默認）
    • 實時傳輸協議（RTP）
    • 安全可靠的傳輸（SRT）
  • RTMP IP白名單 -輸入一組允許連接到RTMP端點的IP地址。這控制了誰可以連接到RTMP端點。請注意，僅當輸入格式為RTMP時，此選項才會出現。
  • CIDR白名單  -輸入一組允許連接到RTP或SRT端點的IP地址。這可以控制誰可以連接到RTP或SRT端點。請注意，只有在輸入格式是RTP或SRT。
  • +添加多CDN配置 - 進入一個標籤和網址用於其他CDN。這是CDN的白名單，允許向我們的主要CDN詢問視頻片段。您負責正確配置輔助CDN。

使用控制室查看實時事件信息

“控制室"頁面用於查看有關現場活動的信息。當您創建新的直播事件並開始流式傳輸時，“控制室"頁面將打開，顯示事件信息。有關使用控制室的信息，請參見使用控制室查看實時事件信息。

複製活動

複製事件使創建具有先前活動事件的相同設置的另一個事件變得容易。要復制事件，請按照下列步驟操作：

  1. 單擊一個事件以打開“控制室"頁面。
  2. 點擊重複事件在頁面頂部。將使用先前現場直播的設置創建一個新的現場直播。根據需要更改事件屬性。
  3. 點擊建立活動。

創建一個片段

可以在事件進行中或事件結束後創建剪輯。如果無法創建片段，則將鼠標懸停在上方時會出現一條消息創建剪輯在左側導航欄中。有關創建剪輯的信息，請參見創建視頻剪輯。

注意：直播活動結束後 7 天內可以剪輯視頻流。

對直播問題進行故障排除

如果您需要進一步的幫助以使現場活動開始進行，可以聯繫我們。為確保獲得最快的響應，下面列出了解決此問題所需的支持。

  • 流所具有的特定症狀。例如，它根本不播放還是停頓或凍結？
  • 過去該流是否正常工作
  • 您在編碼器中使用的入口點URL
  • 您正在使用的編碼軟件和硬件
  • Brightcove Engage 中實時資產的視頻 ID ™
  • 從編碼器到發佈點主機的跟踪路由的結果
