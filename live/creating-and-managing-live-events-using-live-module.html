    建立與管理即時事件

    在本主題中，您將學習如何建立和管理即時活動。

    簡介

    Brightcove Ent習™ 中的即時視頻功能可用於將現場活動轉播給行動應用程式和網路體驗。建立現場活動後，您將會收到一組編碼器設定，您可以將其插入現場編碼器。所有轉碼都在雲端完成，因此您不需要基於硬件的轉碼器。然後，即可在建立行動應用程式和網路體驗時加入即時影片。

    先決條件

    在廣播即時活動之前，必須先提供下列項目：

    • 播放活動的攝影機
    • 您必須擁有自己的編碼軟體，例如電信的 Wirecast 或開放式廣播軟體

    限制

    • 直播將經歷 20-30 秒的延遲
    • 串流必須在活動開始後的 30 分鐘內開始
    • 對於使用網際網路總管的使用者，需要版本 10 或更新版本

    管理即時事件

    要開始直播活動，請登錄Brightcove Engage™然後點擊生活在導航標題中。「管理事件」頁面會顯示兩個主要標籤：

    • Live -列出進行中的即時事件
    • 已完成 -列出已完成的事件

    [ 顯示 ] 下拉式清單可用來篩選出現在 [ 已完成 ] 索引標籤上的事件。

    頁面的主體將顯示具有相關資訊的即時事件清單：

    • 工作狀態-即時串流的狀態
      • 處理 -串流即時
      • 處理錯誤 -即時串流發生錯誤
      • 待命 -即將啟動的串流 (僅限靜態進入點)
      • 停止 -即時流正在停止並創建任何相關的 VOD 輸出
      • 已完成 -即時串流已完成
      • 取消 -即時串流正在停止，不會處理任何待定的 VOD 輸出
      • 已取消 -直播已取消
      • 中斷連線 -編碼器已中斷連線
      • 等待 -等待 RTMP 串流 (編碼器啟動)
      • 失敗 -工作因系統錯誤而停止
      • 已放棄 -事件在沒有連接編碼器的情況下完成
    • 完成的日期和時間
    • 事件名稱和識別碼
    • 總檢視
    • 持續時間
    • 建立的日期和時間 (僅適用於週期性事件)

    若要檢視即時事件視訊屬性，請按一下事件名稱以開啟控制室。

    建立新事件

    若要建立新的直播活動，請依照下列步驟執行。

    1. 從「管理事件」頁面按一下「建立事件」。「建立即時活動」頁面隨即開啟。
    2. 輸入活動詳細資料：
      • 活動名稱 -活動名稱；如果將視頻另存為視頻資產，此名稱也將出現在“視頻”頁面中（最多255個字符）
      • 簡短說明 -事件的摘要 (最多 250 個字元)
      • 標籤 -要與事件關聯的標籤；如果視訊儲存為視訊資產 (最多 128 個字元和 1200 個標籤)，則會將標籤指派給視訊
      • 參考 ID -事件的參考 ID；必須是唯一的 (最多 150 個字元)
    3. 選擇完成後將事件轉換為視頻資產將實時視頻的視頻點播版本保存到您的帳戶。請注意，直播活動結束後將無法使用VOD版本，上傳過程完成後，視頻將可用。
    4. 在裡面流選項部分，選擇一個地區。的地區是要使用的Amazon Web Services（AWS）區域。若要減少延遲，您應該選擇最接近編碼器所在位置的區域。默認情況下，該地區設置為俄勒岡州，如果您是從其他大洲流媒體，則會產生不好的結果。支援下列區域：
      • 俄勒岡州（預設值）
      • 維吉尼亞
      • 東京
      • 新加坡
      • 悉尼
      • 孟買
      • 法蘭克福
      • 愛爾蘭
    5. 按一下「進階選項」並設定進階選項。
    6. 按一下「建立事件」以啟動即時事件。[ 控制室] 頁面將會開啟，其中包含即時事件的詳細資料，包括編碼器將使用的串流端點 (RTMP URL) 和串流名稱。
      注意：處理串流需要大約一分鐘的時間。

      根據預設，編碼器必須在建立新事件後的 30 分鐘內啟動。

    按一下「停止串流」按鈕以結束事件。

    注意：確保編碼器在即時串流完成時關閉/停止。在任何編碼器和 Brightcove 之間保持連線開啟，即使沒有傳送任何資料，也可能會導致未預期的連線費用。

    設定進階選項

    建立新事件時，可以設定進階選項。若要設定進階選項，請在建立事件時按一下 [ 進階選項 ] 連結。

    您可以設定下列選項：

    • 內容加密 -選取要套用至即時串流的加密類型：
      • 無 -未套用加密
      • AES-128 -使用 AES-128 加密來保護資料流的安全。如果選取此選項，您必須輸入加密金鑰。金鑰必須是 32 個十六進位字串。為獲得最佳結果，請使用金鑰產生器，選擇128-bit和Hex選項。
        剪輯片段無法從加密的即時串流建立。
    • 輸入格式 -來自編碼器的輸入格式，將被發送到Brightcove Live。此功能可在現在直播中提供額外的中繼資料，例如現在-下一步，在字幕和字幕、音軌方面更具彈性，並透過標準的廣播層級資料提供簡化的數位體驗。支援下列輸入格式：
      • RTMP (預設值)
      • 即時傳輸通訊協定 (RTP)
      • 安全可靠的傳輸 (SRT)
    • RTMP IP 白名單 -輸入一組允許連線至 RTMP 端點的 IP 位址。這會控制誰可以連線到 RTMP 端點。注意只有在輸入格式為 RTMP 時，才會出現此選項。
    • CIDR白名單  -輸入一組允許連接到RTP或SRT端點的IP地址。這會控制誰可以連線到 RTP 或 SRT 端點。注意只有在輸入格式為 RTP 或 SRT 時，才會出現此選項。
    • + 新增多個 CDN 設定 -輸入其他 CDN 的標籤和 URL 。這是允許詢問我們的主要 CDN 視頻段的 CDN 的白名單。您必須負責正確設定次要 CDN。

    使用控制室檢視即時活動資訊

    「控制室」頁面可用來檢視即時事件的相關資訊。當您建立新的直播活動並開始串流時，「控制室」頁面會開啟，顯示事件資訊。有關使用控制室的信息，請參閱使用控制室查看實時事件信息。

    複製事件

    複製事件可讓您輕鬆建立另一個事件，並使用相同的設定來建立先前的現場活動。若要複製事件，請依照下列步驟執行：

    1. 按一下事件以開啟 [控制室] 頁面。
    2. 按一下頁面頂端的「複製事件」。將使用先前即時活動的設定來建立新的直播活動。視需要變更事件屬性。
    3. 按一下「建立事件」。

    建立剪輯片段

    您可以在活動進行時或活動結束後建立剪輯片段。如果無法建立剪輯片段，將滑鼠游標懸停在左側導覽中的 [ 建立剪輯片段] 上時，就會出現訊息。如需有關建立剪輯的資訊，請參閱建立視訊片段。

    注意：直播活動結束後，可在 7 天內進行剪輯。

    即時串流問題疑難排解

    如果您需要進一步協助讓您的直播活動正常運作，請與我們聯絡。為了確保您得到最快的回應，以下是解決問題所需的支援清單。

    • 流的具體症狀。例如，它是根本不播放還是口吃或凍結？
    • 這個流是否在過去正常工作
    • 您在編碼器中使用的入口點 URL
    • 您使用的編碼軟件和硬件
    • 您在布萊特灣互動™ 中的即時資產的視訊 ID
    • 從編碼器到發佈點主機的追蹤路由結果
