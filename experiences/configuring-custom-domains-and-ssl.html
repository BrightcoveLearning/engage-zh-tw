    正在設定自訂網域和 SSL

    在本主題中，您將學習如何設定 Web 體驗的自訂網域和 SSL。

    首次創建Web Experience時，會為其分配一個隨機域，其格式為<site-slug>.bcvp0rtal.com 。例如，http://site-148393.bcvp0rtal.com .經驗發佈後，觀眾將能夠訪問該網域的網站。Brightcove 互動提供在布賴特灣提供的網域或您自己的自訂網域上發佈網路體驗的能力。

    網路體驗也可以指派給多個 URL，讓發佈者在不同的功能上使用相同的網站，而不必複製網站。例如，如果您有一個想要在廣告活動中使用的視訊網站，而且做為主要的影片入口網站，您可以將不同的 URL 加以品牌化，並將它們全部指派給同一個網站。不需要網站複製，您可以在不再需要網址時輕鬆刪除。

    使用布賴特灣提供的域名

    Brightcove 提供的域名允許您快速輕鬆地為您的 Web 體驗創建一個域名，而無需與您的 IT 部門合作創建自定義 CNAME 或向註冊商註冊新的域名。它們可以「開箱即用」，只需點擊一個按鈕即可實現。可用的網域包括：

    • 亮麗的影片。視訊
    • bcvp0rtal.com (預設會指派給網路體驗)

    所有這些都支持 SSL。要訪問 SSL 網站，只需將 https 添加到 URL 中即可。例如，https://mysite.brightcoveengage.video

    使用自訂網域 (發行者已註冊)

    網路體驗也可以託管在您自己註冊的自訂網域上。這允許您對網址進行品牌，並保持其與您當前的網站/URL 命名結構一致。這會要求您註冊網域名稱，然後向您的主機供應商設定自訂 CNAME 記錄。如需有關如何設定 CNAME 記錄的資訊，請參閱以下章節。

    建立自訂網域

    若要建立自訂網域，請依照下列步驟執行：

    1. 開放布萊特灣投入
    2. 打開專案頁，然後單擊設定值頁面頂部的鏈接
    3. 在左側導航欄中，點擊自定義域和SSL鏈接。
    4. 按一下「建立自訂網域」。
    5. 選取要使用的網域選項：
      • brightcoveengage.video -網站域的格式為<your domain>.brightcoveengage.video。
      • 自訂 -選取以在您自己的自訂網域上託管站台。選取此選項時，您也必須與您的主機供應商設定 CNAME 記錄。
    6. 輸入網域名稱。系統會顯示一則訊息，指出網域是否有效或無效。如果網域名稱無效，請輸入新的網域名稱。
    7. 如果您使用 Brightcove 提供的網域發佈網站，則可以選擇在此網域上強制執行 SSL 。檢查一下在此域上實施SSL如果查看者轉到非SSL版本的站點，則可以啟用重定向。
    8. 如果您要將網站發佈至自己的自訂網域，您可以選擇使用 SSL 保護網站。勾選 [ 使用 SSL ] 核取方塊，以使用 SSL 保護網站的安全。有關使用SSL的信息，請參見使用SSL保護Web體驗。
    9. 檢查一下啟用重定向將此URL重定向到另一個URL。通常，這只會在使用此網域一段時間後才會檢查。選取此選項時，請輸入重新導向 URL ，此現有 URL 應重新導向至。 勾選 [ 使用永久重新導向 (HTTP 狀態碼 301) ，表示重新導向是永久性的。當搜索引擎蜘蛛在網頁的響應頭發現一個 301 狀態碼，它明白，這個網頁不再存在，並用新的替換索引的 URL。此選項將保留您的舊網頁流量和頁面排名。
    10. 按一下「儲存」。將會顯示已設定的網域清單。
      注意：使用 Brightcove 提供的網域時，SSL 狀態一律為「提供」。這表明通過添加https：//到站點URL的開頭，您的瀏覽器和Brightcove Engage站點之間的數據傳輸將是安全的。
    11. 視需要建立其他網域。

    要編輯或刪除域，請點擊修改（）或刪除（） 鈕扣。

    建立自訂網域之後，即可將這些網域指派給體驗屬性的一部分。有關信息，請參見將自定義域分配給Web體驗。

    與您的主機供應商配置 CNAME 記錄

    注意：只有在將自訂網域指派給網站時，才需要設定 CNAME 記錄。CNAME 記錄不一定要為使用 Brightcove 提供的網域的網站建立。

    設定 CNAME (標準名稱) 記錄的步驟會因您的主機供應商而有所不同。您需要為每個要使用的自訂網域設定 CNAME 記錄。如果您的網站設定為使用 SSL，CNAME 記錄會指向一個 CloudFront URL。看到使用SSL保護Web體驗有關信息。請注意，自定義域必須包含一個子域才能配置為CNAME ex：engage 必須放在brightcovelearning.com 這樣我就可以配置我的CNAME。

    CNAME 值會顯示為「自訂網域與 SSL」設定的一部分。

    在此示例中，對於engage.brightcovelearning.com，我會添加一個CNAME記錄並添加主機engage 並指向 5e84f4c7d58950766328a37d.bcvp0rtal.com。

    您可能需要聯絡您的網路管理員或網站託管服務供應商，以取得此步驟的協助。請記住，DNS 變更最多可能需要 48 小時才會生效。

    使用 SSL 保護網站

    注意：使用 SSL 憑證時，請勿在任何情況下共用您的私密金鑰或將其傳送至 Brightcove 支援中心。

     使用 SSL 保護入口網站體驗有兩個選項：

    1. 對於使用分配的站點段發布的站點（即<site-slug>.bcvp0rtal.com）或正在使用Brightcove提供的域（<site>.brightcoveengage.video），無需進行配置即可支持https。新增中https：//網址的開頭將確保您的瀏覽器和體驗之間的數據傳輸。您也可以選取 [ 在此網域上強制執行 SSL] (建議使用) ，以便在檢視者前往網站時啟用非 SSL 版本的網站重新導向。
    2. 對於託管在自定義域中的網站，請檢查使用SSL輸入您的域名時的選項。接著，系統會提示您輸入憑證資訊。有關更多信息，請參見使用SSL保護Web體驗。
