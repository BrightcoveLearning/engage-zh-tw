創建訪問控製配置文件

在本主題中，您將學習如何創建訪問控製配置文件。

默認情況下，當 Brightcove 參與™經驗已發布，經驗是公開的：

  • 行動應用程式 -擁有訪問密鑰的任何人都可以訪問該應用
  • 網絡應用 -具有站點URL的任何人都將能夠訪問Web門戶

可以創建訪問控製配置文件，以便在用戶可以訪問應用程序/網站之前，將強制用戶使用有效的憑據進行身份驗證。Brightcove Engage支持安全斷言標記語言2.0（SAML 2.0）標準，用於在安全域之間交換身份驗證和授權數據。

Brightcove Engage支持大多數主要的身份提供者，包括：

  • 平
  • 一次登入
  • 驗證碼0
  • 地址 2.0/3.0
  • 谷歌 G 套件
  • 希伯雷斯
  • 奧克塔
  • 銷售專業網

使用 SAML 2.0 標準的其他提供者也可以運作。

筆記：Brightcove Engage需要SAML 2.0才能進行單點登錄身份驗證。不支持 SAML 1.1。

配置SSO軟件時，請映射以下屬性：

  • firstName映射到名字
  • lastName映射到姓氏
  • 電子郵件映射到電子郵件

創建一個新的個人資料

在項目設置中創建訪問控製配置文件，然後將其分配給體驗。

若要進行訪問控製配置文件，請按照下列步驟操作：

  1. 登入布萊特灣參與。
  2. 來自專案頁面上，單擊設定值。
  3. 在左側導航中，單擊訪問控製配置文件。將顯示“訪問控製配置文件"頁面，其中包含已創建的配置文件列表（如果有）。
  4. 點擊創建個人資料。
  5. 給個人資料一個名稱。
  6. （可選的）配置一個會話超時。這是用戶在需要重新認證之前可以登錄應用程序的時間。一些注意事項：
    • 您的SSO提供者必須設置為強制重新認證或會話超時小於此超時。如果不是，我們會在超時後重新導向到您的 SSO 提供者，他們將立即發出用戶登錄的響應。
  7. 選擇單點登錄選項。將顯示SSO配置選項。
  8. 如果您具有SAML 2.0元數據的導出文件，請單擊選擇文件然後瀏覽並選擇元數據文件。這將預填充SAML 2.0端點和X.509證書領域。
  9. 輸入一個值SAML 2.0端點（HTTP）。您的 SAML 提供者應該可以使用此值。
  10. 輸入一個值X.509證書。您的 SAML 提供者應該可以使用此值。
  11. （可選）輸入SAML頒發者/實體ID覆蓋。在此輸入值以選擇性地覆寫 SAML 發行者。
  12. （可選）輸入ACS URL覆蓋。在此輸入值以選擇性地覆寫 ACS URL。
  13. （可選）輸入SAML AuthnContext覆蓋。在此輸入值以選擇性地覆寫 SAML 認證內容。
  14. (選擇性) 輸入您要傳遞至外部 JavaScript 的任何 SAML 屬性。
  15. （可選）檢查公開所有屬性公開所有SAML屬性以在外部JavaScript中使用。
  16. 點擊保存。
  17. 驗證您創建的配置文件是否出現在“訪問控製配置文件"列表中。

要編輯或刪除訪問控製配置文件，請單擊編輯 () 或刪除 () 圖標與配置文件關聯。創建訪問控製配置文件後，可以將它們作為站點屬性的一部分分配給站點。有關信息，請參閱控制對體驗的訪問。

配置ACS URL

您的 SSO 提供者將需要判斷提示取用者服務 URL (ACS URL)。ACS URL 會列在「存取控制設定檔」表格的 SSO 段落中。

ACS URL默認為https://auth.bcvp0rtal.com/login/callback。您也可以提供自己的 URL 來覆寫 ACS URL。這兩個選項都可用於設定更嚴格的 SSO 提供者。

注意：使用IDP啟動的SSO工作流程時，在SSO軟件中指定的ACS URL應指向配置有該訪問控製配置文件的Brightcove Engage站點。在這種情況下，https://auth.bcvp0rtal.com/login/callback不應使用 URL。
